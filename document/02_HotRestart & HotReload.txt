1. 컴파일러
    JIT : 앱을 시간이 오래 걸리는 컴파일 과정을 거치지 않고서도 코드의 변경 사항을 즉시 반영
    AOT :

2. Hot Restart
    - 앱을 재시작 하는 형태
    - 앱의 모든 상태가 초기화 되고, 앱의 코드 변경 사항을 반영

3. Hot Reload
    - 앱을 재실행하지는 않지만, 코드의 변경 사항을 반영
    - 앱의 상태를 유지한 상태로, 코드의 수정 사항을 확인하여 더욱 빠른 개발이 가능

4. 주의 사항
    - 두개 모두 앱을 재 컴파일 후 빌드 하는 것이 아님
    - 앱의 기초가 되는 네이티브 코드 수정, 라이브러리나 Asset 의 추가 등의 상황에는 정상적으로 동작하지 않음
    - 지나치게 많은 코드 변경사항이 존재하는 경우 Reload-Restart 순으로 실행중인 프로세스가 상위 재시동을 요청가능
    - 추후 상태를 관리하는 개발단계에 도달하게 되면 해당 기능 사용에 유의
